# 42.15 Changelog applicability

Summary of how the official **Modding Changelog (42.15)** applies to this mod.  
Full text: `reference/general/Modding Changelog (42.15).pdf` (extract with `pdftotext` to read).

## No change needed

- **Mod ID format** – This mod does not use `require`, `incompatible`, `loadModAfter`, or `loadModBefore` in `mod.info`. The new MOD_ID-only format (no `WORKSHOP_ID\MOD_ID`) does not require any update here.
- **Mod folder layout** – The mod already uses a single versioned subfolder (`42.13`) containing `mod.info`; 42.15’s “only 1 subfolder with mod.info” rule is satisfied.
- **poster/icon** – Uses `poster.png`; the Linux filename bug fix in 42.15 mainly affects names like `icon.png`.

## Optional / when targeting 42.15

- **Translations** – 42.15 uses **.json** translation files in UTF-8; **.txt** translation files are no longer supported.  
  EN has been converted: `media/lua/shared/Translate/EN/Sandbox.json` (same three keys); the old `Sandbox_EN.txt` has been removed. For structure reference see **reference/general/Ex-Translation/** (`EN/Mod.json`, `EN/UI.json`). To add more languages, add a language folder (e.g. `RU/`) with a `Sandbox.json` in the same key-value format.
- **mod.info name/description** – 42.15 supports translating mod name and description via a `Mod.json` file in each language folder (see Ex-Translation). Optional.

## Not relevant to this mod

- Translation-only mods, .glb file watcher, Workshop preview image size, Mod Manager panel/source detection, server mod-connection fixes – no direct impact on current behavior.
